How to use the app and what does the diagram represent?

ICSD is a web program that reads a programming code function and draws a diagram, which allows to learn about the function's structure and to detect duplicate / clone code visually.
Please insert a function in the homepage, or you can try the test function for first impression, and then click the convert button.

As you can see at the top of the page (legend), each control statement is represented by a unique shape. for example, a for loop is represented by a blue circle, a if statement is represented by a yellow trapezoid.

Please read the diagram from left to right and from top to bottom. If there is a shape under a shape connected with a line, that means that there is a nesting. for example, If there is a one blue circle, and below another blue circle, it means there is a for loop inside a for loop. The size of a shape indicates how big the control statement relative to the shape that above.
The space between shapes indicates how much lines there is between the two control statements.

If you see two or more similar patterns of shapes, that may indicate about a duplicate code.
You can click on each shape in the diagram and the app will show you what code that shape represents. You can open up to 4 windows in parallel. You can merge shapes into a one code window by holding CTRL and click on the shapes. When there is more than 2 windows code open, you can click on the ratios link and see how similar the codes are.

What is ICSD?

ICSD is an acronyms of Interactive Control Structure Diagram. ICSD is a web program that reads a programming code function, identifies the structure of the function and draws a diagram that describes the function's structure. The diagram allows to learn about the function's structure and to detect duplicate/clone code visually.

What is duplicate code?

Duplicate code / clone code is a code that occurs many times within a program. If the programmer wants to do any change to the code, he will need to remember all the places the code occurs, and then change the code every place that the code occurs. Avoiding duplicate code makes it easier to the programmer to make changes when needed.
